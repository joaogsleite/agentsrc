# Next.js Application

- Use the App Router and keep server components as the default. Add client components only when browser state or browser APIs are necessary.
- Preserve keyboard access, visible focus treatment, semantic landmarks, and responsive behavior for every user-facing change.
- Validate untrusted input at route boundaries. Do not log request bodies, credentials, or personal feedback content.
- Run `npm run lint`, `npm run check`, and `npm run build` for application changes when dependencies are installed.
